SequenceComparer is a Jython-based extension for Burp Suite that enables advanced comparison of sequences of HTTP requests and their responses. This tool is particularly useful for analyzing differences and similarities between two sets of HTTP sequences.

Features

Context Menu Integration

Select one or multiple requests in the Burp Suites Proxy tab, then send them to SequenceComparer via the context menu.

Sequence Overview

Displays sequences in an array format, allowing you to assign names to sequences, view the number of requests in the sequence, and the total length of the responses in the sequence. The sequence order can be reversed.

Sequence Selection and Display

Select and display two sequences simultaneously, with requests from each sequence displayed in separate tables for easy side-by-side analysis.

Longest Common Subsequence (LCS) Analysis

Uses a dynamic programming algorithm to identify the longest common sequence of requests between two selected sequences. Request rows are coded by color to denote uniqueness or commonality in sequences and response body similarities and differences.

Detailed Request/Response View

Select a request to view its details or response body, depending on the selected mode. Utilizes a diff algorithm for a detailed comparison, highlighting deleted, added, and modified content in different colors.

Scroll Synchronization

Optional scroll synchronization feature allows you to scroll through the requests/responses of two sequences simultaneously, with scrolling in one panel mirroring the other.

Auto-Select Mode

Enable auto-select mode to automatically select a requests counterpart in the other sequence (if it exists) when a request is selected.

Usage

1. Select requests from Burp Suites HTTP history or repeater.
2. Right-click and choose Send to SequenceComparer.
3. Open the SequenceComparer tab to analyze sequences:
- Use the dynamic programming-based LCS analysis for comparisons.
- Explore detailed differences with the diff algorithm.
- Adjust settings for scroll synchronization and auto-select mode as needed.
